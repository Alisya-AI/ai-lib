# Amazon S3 Conventions

- Use explicit bucket/key naming conventions and central helper functions for key generation.
- Prefer pre-signed URLs for direct client uploads/downloads instead of proxying large files through app servers.
- Set appropriate object metadata (`content-type`, cache headers, encryption) at write time.
- Enforce least-privilege IAM for bucket access and separate read/write roles when possible.
- Validate file type/size constraints before issuing upload authorization.
- Keep lifecycle/retention policies in infrastructure code, not ad-hoc runtime scripts.
